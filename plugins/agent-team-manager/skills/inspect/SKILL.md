# Agent Team Inspector

에이전트 팀의 구성과 효율성을 체계적으로 점검하여 토큰 낭비 포인트와 절약 기회를 발굴한다.

## 지원 팀 유형

- **AgentTeam** — 소프트웨어 업체 역할의 풀-스택 에이전트 팀 (`team-manifest.md` 기반)
- **ClaudeBuffett** — 투자 분석 에이전트 팀
- **productivity-engineering-team** — 생산성 엔지니어링 에이전트 팀 (`start.md` + `Team/` 기반)
- 그 외 사용자가 지정하는 임의의 에이전트 팀 디렉터리

팀 디렉터리가 명시되지 않은 경우, 현재 작업 디렉터리에서 팀 관련 디렉터리를 탐색한다.

---

## 점검 절차

### 1단계: 팀 구조 파악

다음 파일들을 병렬로 읽는다 (있는 파일만):

- `team-manifest.md` 또는 `Team/members/` — 에이전트 목록·모델 배정
- `CLAUDE.md` 또는 `start.md` — 팀 운영 원칙, PM 구조
- `rules/` 목록 — 규칙 파일 목록 확인 (전체 내용은 다음 단계에서 필요 시 읽음)

파악 후 에이전트를 표로 정리한다:

| 이름 | 역할/부서 | 배정 모델 | 비용 등급 |
|------|---------|---------|---------|
| ... | ... | ... | High / Mid / Low |

**모델 비용 등급 기준:**
- 🔴 **High**: `claude-opus-4-8`, `claude-opus-4-7`, `gpt-5.3-codex`, `gpt-5`
- 🟡 **Mid**: `claude-sonnet-4-6`, `gpt-4.1`
- 🟢 **Low**: `claude-haiku-4-5`, `codex-mini-latest`, `gpt-4.1-mini`

에이전트 수가 20명 이상이면 모든 프로파일 파일을 읽지 않고 manifest 기반으로만 분석한다. (토큰 절약)

### 2단계: 규칙 준수 점검

`rules/token-efficiency.md` 또는 동등한 파일을 읽는다. 없으면 이 단계를 건너뛴다.

확인 항목:
- 토큰 절약 규칙이 명시되어 있는가
- 규칙이 에이전트 프로파일에 실제로 반영되어 있는가 (필요한 파일만 읽도록 지침이 있는가)
- 서브에이전트에 전달할 컨텍스트를 최소화하는 원칙이 있는가

### 3단계: 모델 적합성 분석

각 역할에 배정된 모델이 적합한지 판단한다.

**적합한 모델 배정 기준:**

| 역할 유형 | 권장 등급 | 이유 |
|---------|---------|------|
| 최고 의사결정 (CTO, Chief Architect, Lead) | High | 복잡한 판단, 창의적 설계 |
| PM, 기획 리드, 디자인 리드 | Mid | 의사소통, 문서화, 분석 |
| 일반 개발, QA, 운영 | Mid ~ Low | 반복 작업, 코드 실행 |
| 단순 자동화 (스크립트, 로그 분석) | Low | 빠른 처리, 비용 절감 |
| Motion Designer, 단순 포맷 변환 | Low | 창의성 불필요 |

역할 대비 모델이 1등급 이상 차이나면 낭비 또는 부족으로 표시한다.

### 4단계: 토큰 낭비·절약 포인트 탐지

다음 패턴을 확인한다 (에이전트 프로파일 샘플 1~3개를 읽어 파악):

**낭비 패턴:**

| 패턴 | 확인 방법 | 심각도 |
|------|---------|------|
| 단순 역할에 Opus 배정 | 모델 vs 역할 대조 | 🔴 High |
| 에이전트 프로파일이 500토큰 초과 | 파일 크기 확인 | 🟡 Medium |
| 규칙이 여러 파일에 중복 정의 | rules/ 파일들 대조 | 🟡 Medium |
| 서브에이전트 컨텍스트 전달 지침 없음 | CLAUDE.md 확인 | 🟡 Medium |
| 역할이 겹치는 에이전트 2명 이상 | manifest 분석 | 🟢 Low |
| 위키 전체를 에이전트가 항상 로드 | 에이전트 프로파일 확인 | 🟡 Medium |

**절약 기회:**

| 기회 | 설명 |
|------|------|
| 모델 다운그레이드 | Opus → Sonnet, Sonnet → Haiku 가능한 역할 |
| 역할 통합 | 비슷한 역할 2명을 1명으로 통합 |
| 공통 컨텍스트 추출 | 여러 에이전트가 반복 읽는 파일을 공유 컨텍스트화 |
| Skill/Plugin화 | 3회 이상 반복되는 패턴을 자동화 |
| 에이전트 프로파일 경량화 | 불필요한 배경 설명 제거 |

## 주의사항

- 에이전트가 20명 이상이면 프로파일 파일을 개별로 읽지 않는다 — manifest 기반 분석만 수행한다
- 없는 파일은 건너뛴다 — 점검 항목 전체를 억지로 채우지 않는다
- 분석 결과는 현재 상태 기준이며, 팀 목적·프로젝트 복잡도에 따라 High 모델이 정당할 수 있음을 명시한다
