# qa 스킬 — QA 워크플로우 오케스트레이터

## 호출 시 즉시 선언

이 스킬을 호출하면 반드시 먼저 말한다:
> "네. QA 워크플로우를 시작하겠습니다."

## 입력 파악

인자를 아래 규칙으로 해석한다:

- **인자 없음** → 프로젝트명을 사용자에게 확인한 뒤 파일 경로를 확인한다
- **인자에 `\` 또는 `.md` 포함** → 파일경로로 해석. 프로젝트명은 아래 자동 탐지 규칙 적용
- **인자에 `\`와 `.md` 없음** → 프로젝트명으로 해석. `D:\workspace\{project}\qa\` 에서 가장 최근 `.md` 파일을 Glob으로 탐색

## 프로젝트 자동 탐지 규칙 (우선순위 순)

1. 인자로 프로젝트명이 명시된 경우
2. 현재 작업 디렉토리가 `D:\workspace\{project}\` 하위인 경우 — Bash로 `pwd` 실행 후 경로에서 추출
3. 불명확하면 사용자에게 확인

## 실행 순서

프로젝트명과 파일 경로가 확정되면 아래 순서로 진행한다:

1. **qa-report 스킬 호출** — 리포트 파싱 & 저장
2. **qa-tasks 스킬 호출** — TODO 생성 & 서브에이전트 실행 & PR 생성

각 단계가 완료될 때까지 다음 단계로 넘어가지 않는다.
